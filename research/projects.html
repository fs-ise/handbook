## From ideas to projects

### Ideas

- Research ideas should be formulated in relatively specific concept (markdown) files.
- Stored in a personal or shared repository (such as the [research-hub](https://github.com/fs-ise/research-hub){target=_blank}).
- Yaml header:

```markdown
---
status: concept
...
---

Summary of the idea (in Markdown)...

```
::: {.callout-note title="Example: ideas" collapse=true}
[NeuroPoly](https://intranet.neuro.polymtl.ca/ideas-for-cool-projects.html){target=_blank}
:::

### Selection

- Consider concepts with `status: concept` in personal or shared repositories (such as the [research-hub](https://github.com/fs-ise/research-hub){target=_blank}).

Can be searched in Obsidian:

```markdown
["status":concept]
```

- Consider fit with [research areas](/research/statement.html)
- Consider resources (current project workload, availability of collaborators etc.)

### Creation

::: resource
- Projects are curated in the [data/projects.yml](https://github.com/fs-ise/handbook/blob/main/data/projects.yml){target=_blank}.
- Pages are automatically generated [here](https://github.com/fs-ise/handbook/tree/main/research/projects){target=_blank}.
- All our research repositories on [GitHub](https://github.com/orgs/fs-ise/repositories?q=topic%3Aresearch){target=_blank}.
:::

## Project management

### Access

To give access to a repository, navigate to the repository settings (collaborators).

### Documentation

Notes on project management are added to the README.md file of the repository.
Expected contributions can be specified using [CRediT](https://credit.niso.org/){target=_blank}.
The following can be used as a template:

::: callout-note
```markdown
**Research project title (abbreviation)**

Target: Journal A, B

Contributors:

- Contributor A (initials): Conceptualization, Writing - original draft, Writing - review & editing, Supervision
- Contributor B (initials): Data Curation, Writing - review & editing

Instructions for setup and building the paper:

make pdf

## Log

Next steps:

- [ ] (initials): Taks A
- [ ] (initials): Task B

## CW1

- [X] Task C
```
:::
